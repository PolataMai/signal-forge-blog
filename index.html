Method Flow

从输入知识，到输出资产。

我给这类博客定的标准不是“写了多少”，而是“能不能复用”。 所以内容会尽量遵循固定流转：问题背景、关键约束、可执行步骤、失败点和模板片段。

Capture

收集碎片

把命令、提示词、异常栈、部署操作先收进一个临时池。

Distill

抽出模式

提炼哪些做法是可复用的，哪些只是一次性的环境偶然性。

Forge

封装条目

形成文章、模板、脚本或 skill，确保下一次可以直接拿来用。

Deploy Ready

这套博客可以直接丢到你的服务器上。

站点没有构建步骤，也没有依赖安装。上传整个目录到服务器静态站点根目录， 用 Nginx 或 Caddy 指向它即可。

$ scp -r ./person-blog user@your-server:/var/www/signal-forge
$ sudo ln -s /etc/nginx/sites-available/signal-forge.conf
$ sudo systemctl reload nginx
